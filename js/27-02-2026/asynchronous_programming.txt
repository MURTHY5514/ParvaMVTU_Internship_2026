Asynchronous Programming in JavaScript – Full Guide

Asynchronous programming allows JavaScript to run tasks without blocking the main thread. This is very important for operations like API calls, database requests, file loading, timers, etc.

JavaScript is single-threaded, but asynchronous behavior is handled using the Event Loop.

1. What is Asynchronous Programming?

Synchronous code runs line by line.

console.log("Start");
console.log("Middle");
console.log("End");

Output

Start
Middle
End

Asynchronous code does not wait for some tasks to finish.

console.log("Start");

setTimeout(() => {
  console.log("Inside Timeout");
}, 2000);

console.log("End");

Output

Start
End
Inside Timeout

Here setTimeout runs after 2 seconds without blocking the program.

2. JavaScript Execution Model

JavaScript asynchronous behavior works using:

Call Stack

Web APIs

Callback Queue

Event Loop

Flow:

Call Stack → Web APIs → Callback Queue → Event Loop → Call Stack

Example:

console.log("Start");

setTimeout(() => {
  console.log("Async Task");
}, 2000);

console.log("End");

Steps

Start goes to call stack

setTimeout moves to Web API

After 2 sec → callback queue

Event loop pushes it to call stack

Output

Start
End
Async Task
3. Callbacks

A callback is a function passed as an argument to another function.

Example
function greet(name, callback) {
    console.log("Hello " + name);
    callback();
}

function sayBye() {
    console.log("Goodbye!");
}

greet("Murthy", sayBye);

Output

Hello Murthy
Goodbye!
Callback with async task
setTimeout(function () {
    console.log("Task completed");
}, 2000);
Callback Hell Problem

When callbacks are nested deeply.

setTimeout(() => {
    console.log("Step 1");

    setTimeout(() => {
        console.log("Step 2");

        setTimeout(() => {
            console.log("Step 3");
        }, 1000);

    }, 1000);

}, 1000);

Problems:

Hard to read

Hard to debug

Hard to maintain

Solution → Promises

4. Promises

A Promise represents a value that may be available now, later, or never.

Promise States

Pending

Resolved (fulfilled)

Rejected

Basic Promise Example
let promise = new Promise((resolve, reject) => {

    let success = true;

    if (success) {
        resolve("Operation successful");
    } else {
        reject("Operation failed");
    }

});

promise
.then(result => console.log(result))
.catch(error => console.log(error));
Promise with setTimeout
function fetchData() {
    return new Promise((resolve) => {

        setTimeout(() => {
            resolve("Data received");
        }, 2000);

    });
}

fetchData().then(data => console.log(data));

Output after 2 sec:

Data received
5. Promise Chaining
function step1() {
    return Promise.resolve("Step 1 completed");
}

function step2() {
    return Promise.resolve("Step 2 completed");
}

step1()
.then(result => {
    console.log(result);
    return step2();
})
.then(result => {
    console.log(result);
});

Output

Step 1 completed
Step 2 completed
6. Async / Await (Modern Approach)

async/await makes asynchronous code look like synchronous code.

Example
function fetchData() {

    return new Promise(resolve => {
        setTimeout(() => {
            resolve("Data loaded");
        }, 2000);
    });

}

async function getData() {
    let result = await fetchData();
    console.log(result);
}

getData();

Output

Data loaded
Error Handling with async/await
async function test() {

    try {
        let result = await fetch("https://api.example.com/data");
        console.log(result);
    } catch (error) {
        console.log("Error:", error);
    }

}

test();
7. Fetch API (Real World Example)

Fetching data from API.

async function getUsers() {

    let response = await fetch("https://jsonplaceholder.typicode.com/users");

    let data = await response.json();

    console.log(data);

}

getUsers();
